פרק 1

מה זה Git ו-GitHub?

הבנת ההבדל בין כלי הגרסאות לפלטפורמת השיתוף

🗂️

Git

מערכת בקרת גרסאות מבוזרת שרצה על המחשב שלך. Git עוקב אחרי כל שינוי בקוד, מאפשר לחזור לגרסאות קודמות ולעבוד במקביל על ענפים שונים.

  • ✔ עבודה אופליין
  • ✔ ניהול גרסאות מקומי
  • ✔ branching מהיר
🐙

GitHub

פלטפורמה מבוססת ענן לאחסון ושיתוף קוד המשתמשת ב-Git. GitHub מוסיפה כלים לשיתוף פעולה כמו Pull Requests, Issues ו-Actions.

  • ✔ שיתוף עם הצוות
  • ✔ Code Review
  • ✔ CI/CD אוטומטי
🔄

הקשר ביניהם

Git הוא המנוע, GitHub הוא המכונית השלמה. אפשר להשתמש ב-Git בלי GitHub, אבל GitHub תמיד מסתמכת על Git.

Local → push → GitHub → pull → Team
פרק 2

פקודות Git חיוניות

לחץ על כל פקודה כדי לראות הסבר מפורט ודוגמה

git init בסיסי

יוצר repository חדש בתיקייה הנוכחית

$ git init my-project
Initialized empty Git repository in ~/my-project/.git/
git clone בסיסי

מעתיק repository קיים מ-GitHub

$ git clone https://github.com/user/repo.git
git add בסיסי

מוסיף קבצים ל-staging area לפני commit

$ git add .          # הכל
$ git add index.html  # קובץ ספציפי
git commit בסיסי

שומר snapshot של השינויים עם הודעה תיאורית

$ git commit -m "Add login feature"
git status בסיסי

מציג את מצב הקבצים — מה השתנה, מה staged

$ git status
On branch main
Changes not staged for commit:
  modified: index.html
git log בסיסי

מציג היסטוריית commits

$ git log --oneline
a3f1c2b Add login feature
9e2d4a1 Initial commit
git branch ענפים

יוצר, מציג או מוחק ענפים

$ git branch feature/login
$ git branch -a  # כל הענפים
git checkout / switch ענפים

עובר לענף אחר

$ git switch feature/login
$ git checkout -b new-branch
git merge ענפים

ממזג ענף אחד לתוך הענף הנוכחי

$ git switch main
$ git merge feature/login
git push Remote

מעלה commits ל-GitHub

$ git push origin main
$ git push -u origin feature/login
git pull Remote

מוריד ומשלב שינויים מ-GitHub

$ git pull origin main
git fetch Remote

מוריד שינויים בלי לשלב אוטומטית

$ git fetch origin
$ git diff main origin/main
git rebase מתקדם

מעביר commits לבסיס חדש לשמירה על היסטוריה נקייה

$ git rebase main
$ git rebase -i HEAD~3  # interactive
git stash מתקדם

שומר שינויים זמנית בלי commit

$ git stash
$ git stash pop  # שחזור
git reset מתקדם

מבטל commits או unstage קבצים

$ git reset --soft HEAD~1  # שמור שינויים
$ git reset --hard HEAD~1  # מחק הכל
פרק 3

תהליך עבודה מקצועי

GitHub Flow — השיטה שמשתמשים בה בחברות הגדולות בעולם

01

צור ענף חדש

תמיד עבוד על ענף נפרד, לא ישירות על main

git switch -c feature/awesome-feature
02

Commit שינויים

כתוב הודעות commit ברורות ותיאוריות

git add .
git commit -m "feat: add user authentication"
03

Push לענף ב-GitHub

העלה את הענף שלך לשרת המרוחק

git push -u origin feature/awesome-feature
04

פתח Pull Request

ב-GitHub, פתח PR לבקשת Code Review מחברי הצוות

# ב-GitHub.com: New Pull Request →
# base: main ← compare: feature/awesome-feature
05

Code Review ומיזוג

אחרי אישור — מזג לתוך main ומחק את הענף

git switch main
git merge feature/awesome-feature
git branch -d feature/awesome-feature
